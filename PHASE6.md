# 第六階段升級版

第六階段目標是把既有台股盤後題材操盤系統，升級成更完整的 AI 股票研究與自動化操盤實驗平台。這一階段採用「新增獨立檔案、小幅接入既有架構」的方式，避免重寫網站。

## 新增能力

### 策略中心

集中管理策略定義、啟用狀態、風險規則、近期命中股票與簡易回測摘要。初始策略包含：

- HMA/STDEV
- 布林通道突破 R1
- 箱型突破
- 量縮回測均線糾結
- MA5 強勢股
- 月線支撐
- MACD 轉強

### 型態辨識引擎

以日 K、成交量與均線資料辨識常見型態，輸出支撐、壓力、突破價、停損、目標價與風險說明。

支援型態包含箱型整理、三角收斂、上升三角形、下降三角形、旗形整理、W 底、M 頭、頭肩底、頭肩頂、假突破、放量突破、量縮回測。

### 量價結構評分

在每日訊號中加入量價結構評分，判斷價漲量增、價漲量縮、價跌量增、價跌量縮、低檔爆量、高檔爆量長上影、突破放量與回測量縮。

輸出欄位：

- `volume_price_score`
- `volume_price_tags`
- `risk_flags`

### 主力行為偵測

偵測疑似洗盤、誘多、出貨與主升段前整理行為，包含假跌破洗盤、橫盤磨人洗盤、急殺長黑洗盤、拉高後快速回落、假突破誘多、高檔爆量出貨、突破後回測成功、主升段前整理。

### AI 操盤員檢討日誌

讀取 AI 模擬交易紀錄，自動產生交易檢討，分類進場太晚、追高、假突破、量能不足、大盤環境不佳、題材退燒、停損太慢、停利太早與策略有效。

### 每日自動報告

整合市場總結、強勢題材、資金流入產業、突破觀察股、回測支撐觀察股、高風險警告股、AI 模擬操盤員今日動作與明日觀察重點，寫入 Supabase 並由前端讀取。

- 主力行為

- 每日報告增強版

## 新增資料表

請先在 Supabase SQL Editor 執行 `db/phase6_schema.sql`。此檔案可重複執行，使用 `create table if not exists` 與 `alter table ... add column if not exists`。

## 執行順序

GitHub Actions 每日資料管線在原流程後追加：

1. `compute_patterns.py`
2. `mainforce_detector.py`
3. `strategy_center.py`
4. `ai_trade_journal.py`
5. `generate_daily_report.py`

每個步驟使用 `if: always()`，避免單一分析步驟失敗造成後續不執行。
